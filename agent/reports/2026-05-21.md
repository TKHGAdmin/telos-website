## BUG-2026-05-21-001 — Product detail page layout is broken on desktop (3 grid children in 2-column grid)

- **Severity:** P1
- **Area:** `product.html` (Shop / product detail page)
- **Introduced in:** commit `f5cb8c4` — "feat: add image gallery with thumbnails on product page"
- **Affected users:** Every desktop visitor who clicks into any product detail page (≥768px viewport).

### What's wrong

`.product-detail.loaded` is `display: grid; grid-template-columns: 1fr 1fr` (product.html:105-108), but the container has **three** direct children:

```html
<div class="product-detail" id="productDetail">              <!-- product.html:765 -->
  <div class="product-detail-image" id="productImage"></div>
  <div class="product-detail-thumbs" id="productThumbs"></div>
  <div class="product-detail-info"> ... title, price, button, description ... </div>
</div>
```

CSS Grid auto-flows those three items as:

| | col 1 (1fr) | col 2 (1fr) |
|---|---|---|
| row 1 | `#productImage` | `#productThumbs` |
| row 2 | `.product-detail-info` | *(empty)* |

So the thumbnail strip renders in the right column at full 1fr width (where the title/price/buy button should be), and the entire product info column wraps below the main image, occupying only the left half of the page with a wasted empty cell to its right. The thumbs CSS (`product.html:120-124`) — `display: flex; gap: 10px; margin-top: 12px` — was clearly written assuming they sit *under* the main image, not as a sibling grid track.

### Repro

1. Have any Shopify product configured (any handle).
2. Visit `https://www.telosathleticclub.com/product?p=<handle>` on a desktop viewport.
3. Expected: image on the left, info on the right, small thumbnail row under the image.
4. Actual: image top-left, thumbs top-right (filling the info column), title + price + Add-to-Cart wrapped below the image with a blank gap to their right.

The mobile breakpoint (`@media (max-width: 768px)` at product.html:110-115) collapses to single-column, so phones render in source order (image → thumbs → info) — that case is fine.

### Suggested fix

Cheapest: wrap image + thumbs in one column:

```html
<div class="product-detail" id="productDetail">
  <div class="product-detail-gallery">
    <div class="product-detail-image" id="productImage"></div>
    <div class="product-detail-thumbs" id="productThumbs"></div>
  </div>
  <div class="product-detail-info"> ... </div>
</div>
```

Alternative: give `.product-detail-thumbs` an explicit `grid-column: 1; grid-row: 2;` and put `.product-detail-info` at `grid-column: 2; grid-row: 1 / span 2;`.

- **Introduced in:** commit `9319f9d` — "refactor: move Resources and FAQ into Tools dropdown" (refactor was only applied to `index.html`).

But `pricing.html`, `product.html`, and `shop.html` mobile menus are still flat and only contain System / Charleston / Protein Calc / Hyrox Predictor / Shop / Client Login / Book a Call. `Resources` and `FAQ` are absent. A mobile user on those pages has no way to reach `/resources` or the FAQ anchor from the navigation. Cross-check:

## BUG-2026-05-21-003 — `js/shop.js` cache-buster mismatch between shop.html and product.html

- **Severity:** P3
- **Area:** `shop.html:841`, `product.html:822`
- **Introduced in:** commit `21cc926` / `f5cb8c4` — adding product detail page logic and image gallery (which updated `shop.js` but only bumped the version on `product.html`).

### What's wrong

```
shop.html    : <script src="js/shop.js?v=1">
product.html : <script src="js/shop.js?v=2">
```

Returning visitors who previously loaded `shop.html` while `shop.js` was at `?v=1` keep getting the cached older file when they revisit `shop.html`. The browser treats `shop.js?v=1` and `shop.js?v=2` as separate cache keys, so further changes to `shop.js` only invalidate the cached copy on `product.html`. Since `shop.js` now contains gallery/detail code that's also wired into the cart drawer logic shared by the shop catalog, any future shop.js change that isn't accompanied by a shop.html version bump will silently ship to product visitors only.

Compare with `style.css?v=16` which is consistent across every page.

### Suggested fix

Bump `shop.html:841` to `js/shop.js?v=2` (matching `product.html`). Going forward, treat shop.js the same way style.css is treated — bump every page that loads it on every change. Per `CLAUDE.md`'s "CSS cache busting" convention, the same rule should apply to shared JS.
