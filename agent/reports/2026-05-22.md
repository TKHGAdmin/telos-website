## Bugs

### BUG-20260522-01 — Product detail page layout broken: info column drops below image

- **Severity**: P1
- **Area**: shop
- **File(s)**:
  - `product.html:765-780` (DOM structure)
  - `product.html:100-115` (grid CSS)
- **Status**: new

**What's wrong**

On `/product?p=<handle>`, the two-column layout (image on the left, title/price/Add-to-Cart/description on the right) does not render as designed. Instead, the right column is empty and the product info section appears below the image, taking only the left half of the page. The right half of the page below the image is blank white space. When a product has multiple images, the thumbnail strip renders in the upper-right where the title was supposed to be.

This is a regression from commit `f5cb8c4` ("feat: add image gallery with thumbnails on product page"). That commit added a third direct grid child (`#productThumbs`) without updating the grid template, breaking the previously-working two-column layout.

**Reproduction**

Code trace:

1. `product.html:105-108` defines the layout:
   ```css
   .product-detail.loaded {
     display: grid;
     grid-template-columns: 1fr 1fr;
   }
   ```
2. `product.html:765-780` places **three** direct children inside `#productDetail`:
   ```html
   <div class="product-detail" id="productDetail">
     <div class="product-detail-image" id="productImage"></div>
     <div class="product-detail-thumbs" id="productThumbs"></div>
     <div class="product-detail-info">...</div>
   </div>
   ```
3. With three items in a 2-column grid using default auto-flow, items land at:
   - `productImage` -> row 1, col 1
   - `productThumbs` -> row 1, col 2
   - `productDetail-info` -> row 2, col 1 (col 2 of row 2 stays empty)
4. Live verification: open any product on the site (e.g. `https://www.telosathleticclub.com/product?p=<any-handle>`). The "Add to Cart" button and description appear under the image rather than beside it, and the right column is blank.

**Impact**

Every visitor browsing a product detail page sees a broken layout. The Add-to-Cart button, price, and description are still functional, but their position is wrong and the page looks unfinished. With multiple product images, the thumbnail strip is also displaced (it appears in the top-right corner instead of below the main image). Affects shop conversion on a brand-new revenue feature.

**Suggested fix**

Wrap the image and thumbs in a single grid cell so the info column is preserved. Minimal patch:

```html
<div class="product-detail" id="productDetail">
  <div class="product-detail-media">
    <div class="product-detail-image" id="productImage"></div>
    <div class="product-detail-thumbs" id="productThumbs"></div>
  </div>
  <div class="product-detail-info">
    ...
  </div>
</div>
```

No CSS change required if `.product-detail-media` is a plain block — it will fill column 1 of row 1 and contain both the image and the thumbs strip beneath it. Alternatively, set `grid-column: 1 / -1` on `#productThumbs` and `grid-row: 1 / span 2` on `#productImage` and `#productDetail-info`, but the wrapper approach is simpler.
